# AI Commit Helper 使用说明

这是一款 IntelliJ IDEA 插件，用来根据你在 Commit 窗口中勾选的代码改动，自动生成 Git commit message，并回填到提交信息输入框。

简单说：你勾选要提交的文件，点一下按钮，插件帮你写提交说明。

## 一、先准备什么

使用前需要准备：

1. IntelliJ IDEA。
2. 一个可用的 DeepSeek API Key，或其他兼容 OpenAI Chat Completions 接口的服务。
3. 已安装好的 `AI Commit Helper` 插件 zip。

如果你还没有插件 zip，可以让开发人员打包，产物一般是：

```text
target/ai-commit-helper-1.0.0-SNAPSHOT.zip
```

## 二、安装插件

1. 打开 IDEA。
2. 进入 `Settings | Plugins`。
3. 点击插件页面右上角的齿轮按钮。
4. 选择 `Install Plugin from Disk...`。
5. 选择 `ai-commit-helper-1.0.0-SNAPSHOT.zip`。
6. 按提示重启 IDEA。

重启后，插件就安装完成了。

## 三、配置插件

进入配置页面：

```text
Settings | Tools | AI Commit Helper
```

你会看到这些配置项。

### 1. Base URL

默认值：

```text
https://api.deepseek.com
```

如果你用的是 DeepSeek 官方接口，保持默认即可。

如果你用的是公司内网网关、Ollama、vLLM 或其他兼容 OpenAI 的服务，就填对应服务地址。

注意：这里填基础地址，不要手动加 `/chat/completions`。

正确示例：

```text
https://api.deepseek.com
```

错误示例：

```text
https://api.deepseek.com/chat/completions
```

### 2. Model

填写要使用的模型名称。

默认值：

```text
deepseek-v4-flash
```

如果不确定有哪些模型，可以先填好 API Key，然后点击 `Fetch Models` 按钮。插件会从接口获取模型列表，你可以从下拉框里选择。

模型输入框也支持手动输入，所以即使列表里没有，也可以自己填。

### 3. API Key

填写你的 API Key。

API Key 会保存在 IDEA 的 PasswordSafe 中，不会写到项目代码里。

### 4. Timeout Seconds

连接测试、获取模型等操作的超时时间。

默认值：

```text
30
```

普通使用保持默认即可。

### 5. Max Diff Characters

发送给 AI 的最大代码改动字符数。

默认值：

```text
100000
```

如果你的项目改动很大，可以适当调大；如果担心 token 消耗，可以调小。

### 6. Language

生成 commit message 的语言。

可选：

```text
中文
English
```

默认是中文。

## 四、测试配置是否可用

配置好 `Base URL`、`Model`、`API Key` 后，点击：

```text
Test Connection
```

如果弹出成功提示，说明接口可以连通，模型也可以正常响应。

如果失败，优先检查：

1. API Key 是否正确。
2. Base URL 是否填错。
3. Model 是否存在。
4. 网络是否能访问接口。
5. 公司代理或网关是否拦截请求。

## 五、生成 commit message

1. 打开 IDEA 左侧或底部的 Commit 窗口。
2. 勾选本次要提交的文件。
3. 点击提交信息区域附近的 AI/Git 图标按钮。
4. 插件会显示 `生成中...`。
5. 等待几秒后，生成的 commit message 会自动填入提交信息输入框。

生成期间按钮不能重复点击，避免同时发起多次请求。

插件只会读取当前勾选的改动，不会把没有勾选的文件发送给 AI。

## 六、Commit Message 模板配置

有些公司要求 commit message 必须按固定格式填写，例如：

```text
[修改单编号]T202604205176-1
[缺陷编号]
[修改说明]江苏信托iOS
message概要
1.升级Xcode26以及iOS26适配
```

这个插件支持模板定制。

配置位置仍然是：

```text
Settings | Tools | AI Commit Helper
```

重点看这三个配置：

```text
Message Template Preset
Message Template
Template Variables
```

### 1. Message Template Preset

模板预设。

可选：

```text
Default
Company Numbered
Custom
```

含义：

- `Default`：默认格式，标题加项目符号列表。
- `Company Numbered`：公司编号格式。
- `Custom`：完全自定义格式。

### 2. Message Template

这里写最终 commit message 的格式。

模板里可以使用占位符。

常用占位符：

```text
${title}
```

AI 生成的概要标题。

```text
${items.bullets}
```

AI 生成的改动列表，格式是：

```text
- xxx
- xxx
```

```text
${items.numbered}
```

AI 生成的改动列表，格式是：

```text
1.xxx
2.xxx
```

还可以使用你在 `Template Variables` 中配置的变量，例如：

```text
${changeId}
${bugId}
${description}
${requirementId}
${scope}
```

### 3. Template Variables

这里配置固定字段。

格式是一行一个：

```text
key=value
```

例如：

```text
changeId=T202604205176-1
bugId=
description=江苏信托iOS
```

然后你就可以在 `Message Template` 中这样使用：

```text
[修改单编号]${changeId}
[缺陷编号]${bugId}
[修改说明]${description}
${title}
${items.numbered}
```

最终生成效果类似：

```text
[修改单编号]T202604205176-1
[缺陷编号]
[修改说明]江苏信托iOS
适配 iOS 26 构建环境
1.升级 Xcode 26 相关配置
2.适配 iOS 26 构建兼容性
```

如果某个变量没有配置，插件会把它输出为空，不会把 `${xxx}` 原样留在 commit message 里。

## 七、常见模板示例

### 默认格式

Message Template：

```text
${title}

${items.bullets}
```

生成效果：

```text
优化私信功能

- 调整私信列表展示逻辑
- 修复会话未读状态更新问题
```

### 公司编号格式

Message Template：

```text
[修改单编号]${changeId}
[缺陷编号]${bugId}
[修改说明]${description}
${title}
${items.numbered}
```

Template Variables：

```text
changeId=T202604205176-1
bugId=
description=江苏信托iOS
```

生成效果：

```text
[修改单编号]T202604205176-1
[缺陷编号]
[修改说明]江苏信托iOS
适配 iOS 26 构建环境
1.升级 Xcode 26 相关配置
2.适配 iOS 26 构建兼容性
```

### 自定义需求格式

Message Template：

```text
[需求编号]${requirementId}
[缺陷编号]${bugId}
[影响范围]${scope}

${title}

${items.numbered}
```

Template Variables：

```text
requirementId=REQ-2026-001
bugId=BUG-8848
scope=移动端登录
```

生成效果：

```text
[需求编号]REQ-2026-001
[缺陷编号]BUG-8848
[影响范围]移动端登录

优化登录流程

1.调整验证码校验逻辑
2.补充登录异常提示
```

## 八、常见问题

### 1. 点击按钮没有生成

先检查是否已经勾选了要提交的文件。

如果没有勾选文件，插件不知道要根据哪些改动生成 message。

### 2. 提示 API Key 未配置

进入：

```text
Settings | Tools | AI Commit Helper
```

填写 API Key，然后点击 `Test Connection` 测试。

### 3. Fetch Models 获取不到模型

检查：

1. API Key 是否正确。
2. Base URL 是否正确。
3. 当前服务是否支持 `/models` 接口。
4. 网络是否能访问该服务。

即使获取模型失败，也可以手动输入模型名称。

### 4. 生成结果格式不符合公司要求

检查 `Message Template`。

如果公司要求固定头部字段，就把固定字段写在模板里。

如果字段值每次固定或经常复用，就写到 `Template Variables`。

### 5. 为什么修改单编号和缺陷编号不会自动生成

这些编号通常来自公司流程系统，不一定能从代码 diff 里判断出来。

插件不会让 AI 编造编号。你需要在 `Template Variables` 中配置这些值。

### 6. 会不会泄露没有提交的代码

插件只读取 Commit 窗口中当前勾选的变更。

没有勾选的文件不会被发送给 AI。

## 九、推荐使用流程

第一次使用：

1. 安装插件。
2. 打开 `Settings | Tools | AI Commit Helper`。
3. 填写 Base URL、Model、API Key。
4. 点击 `Test Connection`。
5. 根据公司要求配置 Message Template。
6. 打开 Commit 窗口，勾选文件。
7. 点击 AI/Git 图标生成 commit message。
8. 人工快速检查生成内容。
9. 提交代码。

日常使用：

1. 勾选要提交的文件。
2. 点击 AI/Git 图标。
3. 检查生成的 commit message。
4. 提交。
